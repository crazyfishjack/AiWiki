## 定义

业务理解框架是一套用于客户调研的结构化清单：先判断客户靠什么赚钱、怎么赚钱、遇到什么阻碍，再按主体、价值、流程、组织、技术、环境六个维度收集证据并标注置信度。

---

## 核心公式

```text
业务 =
客户靠什么赚钱（价值层，包括主体和价值） ×
怎么赚的（流程层，包括流程、组织和技术） ×
遇到什么阻碍（压力层，也就是环境）
```

这个公式要求调研者同时回答三类问题：

- 价值层：客户是谁，收入与利润从哪里来。
- 流程层：客户通过什么流程、组织和技术把价值交付出去。
- 压力层：政策、竞争、成本、安全合规和上级要求如何形成阻碍。

---

## 证据与置信度

调研结论必须标注来源。没有实际数据时，不要进入更高阶段的判断。

| 置信度 | 表达方式 | 使用边界 |
|---|---|---|
| 高置信 | "根据[来源]，……" | 有明确、权威或可核验的数据来源 |
| 中置信 | "现有资料显示……但建议确认" | 有资料支撑，但仍需要客户或权威来源确认 |
| 低置信 | "目前没有充分资料支撑，建议核实" | 资料不足，只能作为待验证判断 |
| 无支撑 | "基于现有资料无法回答" | 不得臆想、补全或编造 |

质量红线：严禁臆想和胡编乱造。如果没有实际数据，就不要做更高阶段说明。

## 六个调研维度

### 主体

主体维度回答"客户是谁，以及决策逻辑受什么身份影响"。

| 要素 | 具体内容 |
|---|---|
| 企业性质 | 央企/国企/民企/外资，决定决策逻辑 |
| 股权结构 | 谁是实际控制人、上级主管单位 |
| 主营牌照 | 持有什么经营资质、受哪个部委监管 |
| 规模体量 | 营收/资产/员工人数/子公司数量 |

### 价值

价值维度回答"客户靠什么赚钱"。

| 要素 | 具体内容 |
|---|---|
| 收入来源 | 主营业务收入结构（各板块占比） |
| 盈利模式 | 卖产品/卖服务/卖资源/卖数据 |
| 核心产品 | 旗舰产品是什么，利润从哪来 |
| 客户群体 | B2B / B2G / B2C |

### 流程

流程维度回答"客户怎么把价值交付出去"。

| 要素 | 具体内容 |
|---|---|
| 核心业务流程 | 从原料/线索到最终交付的全链条 |
| 关键作业节点 | 哪些节点是卡脖子/高风险/高成本 |
| 信息流转方式 | 数据在哪里产生、流向哪里、如何使用 |
| 决策机制 | 什么事谁来拍板、依据什么信息 |
| 知识载体 | 经验存在人脑还是文档还是系统里 |

### 组织

组织维度回答"谁推动，谁受影响，谁可能阻碍"。

| 要素 | 具体内容 |
|---|---|
| 部门分工 | 各部门职责边界，谁管生产谁管 IT |
| 人员结构 | 一线操作工/工程师/管理层比例 |
| 能力短板 | 哪类人才稀缺，哪些依赖外包 |
| 利益格局 | 谁是项目的推动者、谁是阻力 |

### 技术

技术维度回答"现有系统和数据能否支撑方案落地"。

| 要素 | 具体内容 |
|---|---|
| 已有系统 | ERP / MES / OA / PLM / SCADA 等 |
| 系统成熟度 | 老系统还是近年上线的 |
| 数据现状 | 数据在哪里、有没有治理、能不能用 |
| 技术偏好 | 是否有信创要求、倾向哪个云 |
| 集成难度 | 各系统是否打通、接口是否标准 |

### 环境

环境维度回答"外部压力如何影响业务优先级"。

| 要素 | 具体内容 |
|---|---|
| 行业政策 | 主管部委的监管要求、十五五政策 |
| 市场竞争 | 同行在做什么，客户的竞争焦虑 |
| 成本压力 | 人力成本上升/原材料波动 |
| 安全合规 | 事故/罚款/审计压力 |
| 上级要求 | 集团下达的数字化 KPI |

## 三档业务理解成熟度

| 段位 | 表现 | 结果 |
|---|---|---|
| 初级 | 能说出客户主营业务是什么 | 只能聊皮毛，例如"他们是做电力设备的" |
| 中级 | 能描述核心业务流程的关键节点 | 能找到痛点，例如"变压器质检现在全靠人工，每班 3 人" |
| 高级 | 能说清痛点背后的组织利益和决策逻辑 | 能推动成交，例如"质检部门不愿上 AI 怕裁员，但生产副总有降本 KPI，这是突破口" |

---

## 使用方法

1. 先用公开资料填充主体、价值和环境维度，避免访谈时反复询问基础事实。
2. 访谈中重点追问流程、组织和技术维度，因为这些维度更容易暴露痛点、阻力和落地条件。
3. 每条结论都标注来源与置信度；如果只停留在低置信或无支撑，就不能用于方案承诺。
4. 用三档成熟度复核调研深度：如果只知道主营业务，就不要急着设计 AI 方案；至少要进入中级后才能稳定识别痛点。
