# 资质管理系统 · 文件水印、无水印审批和安全下载 · 行业对标矩阵

## 1. 调研结论摘要

当前技术思路纪要已经把“权限、水印与下载审批”列为一期 P0 能力：默认文件带服务端水印，无水印下载必须申请并经钉钉审批，审批通过后生成限时下载链接；普通员工不可见跨部门资质，商务/授权用户可查看全量但下载需审批，平台管理员可全量管理与直接下载。此为本地项目资料结论，来源为 `工作区/资质管理系统/2026-06-09-1834-资质管理系统-技术思路纪要.md`，无外部 URL，需在 PRD 评审中继续确认。

行业最佳实践不是只做“PDF 上盖字”，而是组合使用：分类标签、服务端或文档引擎水印、下载/打印/复制限制、限时链接、对象存储签名 URL、审批例外、审计日志、DLP 规则和外发追踪。Microsoft Purview 将敏感度标签用于分类、加密、页眉页脚和水印，并可让 SharePoint 文档库在下载时把权限扩展到文件本身。[来源: https://learn.microsoft.com/en-us/purview/sensitivity-labels] [来源: https://learn.microsoft.com/en-us/purview/sensitivity-labels-sharepoint-onedrive-files]

对象存储的“预签名 URL”适合做短时下载通道，但它本身通常不是“一次性链接”。Amazon S3 官方说明预签名 URL 在过期前可多次使用；若要一次性使用，需要应用层令牌、数据库状态或代理网关兜住。[来源: https://docs.aws.amazon.com/AmazonS3/latest/userguide/using-presigned-url.html] [来源: https://aws.amazon.com/blogs/storage/implement-single-exchange-tokens-for-short-lived-amazon-s3-presigned-urls-with-terraform/]

对当前资质系统，一期建议采用“应用层授权 + 服务端生成带水印副本 + 审批后短时无水印链接 + 全量审计”的保守方案；二期再增强动态水印、DLP/IRM、一次性令牌和异常下载分析。不要把长期公开 URL、前端叠水印、公共读对象桶作为敏感资质文件方案。[来源: https://www.alibabacloud.com/help/en/oss/user-guide/how-to-obtain-the-url-of-a-single-object-or-the-urls-of-multiple-objects] [来源: https://www.tencentcloud.com/document/product/436/45243]

## 2. 当前技术思路纪要中的想法

本地纪要中的 M4 模块可概括为：

| 方向 | 当前想法 | 需要外部对标的问题 |
| --- | --- | --- |
| 权限范围 | 平台管理员、商务用户、授权部门用户、部门资质管理员、普通员工五类角色，多角色取最大授权范围 | 行业内如何避免“可查看但不可下载”被绕过 |
| 默认水印 | 所有资质文件默认带水印，水印由服务端生成，不只在前端叠加 | 水印应包含什么变量，是否要动态水印或不可移除水印 |
| 无水印审批 | 用户申请无水印下载，触发钉钉审批，至少“部门领导审批 → 老板审批”，审批链可配置 | 审批通过后的下载次数、有效期、撤销、审计字段 |
| 限时下载 | 审批通过后生成无水印版文件和限时下载链接 | 预签名 URL 是否足够，是否要一次性链接 |
| 预览与导出 | 带水印文件可随权限导出，无水印 PDF 需审批 | 文件预览和下载是否隔离，预览水印是否能追踪外泄 |
| 集成依赖 | Logto 反查钉钉 ID，复用钉钉网关；API 供投标助手消费 | API 文件能力如何做鉴权、审计、限流和防越权 |

以上为本地项目需求输入，无外部 URL，不能视为行业事实。

## 3. 行业内优秀做法

### 3.1 云文档与企业内容平台

Microsoft Purview 的思路是“标签驱动保护”：敏感度标签可同时承载分类、加密、页眉页脚、水印和使用权限；标签存入文件元数据，可随文件流转。[来源: https://learn.microsoft.com/en-us/purview/sensitivity-labels] SharePoint/OneDrive 启用敏感度标签后，下载文件时标签和加密设置仍被执行并保留，Office/PDF 文件还可进入搜索、DLP、eDiscovery 等治理流程。[来源: https://learn.microsoft.com/en-us/purview/sensitivity-labels-sharepoint-onedrive-files]

Google Workspace Drive DLP 更偏“禁止动作”：管理员可用数据保护规则阻止 Drive 文件下载、打印和复制，并可作用于 My Drive 与共享云端硬盘；但 Google 明确提示该限制不阻止在 Preview 模式中打印文件。[来源: https://knowledge.workspace.google.com/admin/security/dlp-disable-download-print-copy] Google DLP 还支持审计规则、外部分享阻断、告警、调查工具和规则日志。[来源: https://support.google.com/a/answer/9646351]

Box 的做法更接近安全资料室：企业可限制共享链接访问范围、配置共享链接权限为“仅查看”、设置水印、默认共享链接过期时间、外部协作过期时间等。[来源: https://support.box.com/hc/en-us/articles/4404822772755-Enterprise-Settings-Content-Sharing-Tab] Box 水印包含当前查看者姓名和访问时间；栅格化水印安全性更高，但会牺牲文本搜索、复制和可用性。[来源: https://support.box.com/hc/en-us/articles/4404822772755-Enterprise-Settings-Content-Sharing-Tab]

### 3.2 对象存储与安全下载

Amazon S3 预签名 URL 可授予对象临时访问，不改 bucket policy；控制项包括对象 key、HTTP 方法、过期时间和生成者权限。[来源: https://docs.aws.amazon.com/AmazonS3/latest/userguide/using-presigned-url.html] S3 官方说明预签名 URL 在到期前可多次使用，因此“一次性下载”必须用应用层令牌、数据库状态或短时交换机制实现。[来源: https://docs.aws.amazon.com/AmazonS3/latest/userguide/using-presigned-url.html] AWS 还建议用 CloudTrail Data Events 或 S3 Server Access Logs 识别预签名请求，特征包括 `AuthenticationMethod=QueryString` 和 `X-Amz-Expires` 参数。[来源: https://docs.aws.amazon.com/prescriptive-guidance/latest/presigned-url-best-practices/identifying-requests.html]

Azure Storage SAS 用于按资源、权限和有效期委托访问；Microsoft 建议优先使用基于 Microsoft Entra 的 user delegation SAS。[来源: https://learn.microsoft.com/en-us/azure/storage/common/sas-expiration-policy] SAS 过期策略可设置最大有效期，并可选择 Log 或 Block：Log 用于审计而不中断流程，Block 用于严格阻断超策略 SAS。[来源: https://learn.microsoft.com/en-us/azure/storage/common/sas-expiration-policy]

阿里云 OSS 文档强调对象默认私有，可用预签名 URL 给第三方临时下载或预览；URL 有签名和有效期，且在有效期内可多次访问。[来源: https://www.alibabacloud.com/help/en/oss/user-guide/how-to-obtain-the-url-of-a-single-object-or-the-urls-of-multiple-objects] OSS 还明确建议：始终使用 HTTPS、设置尽量短的有效期、使用 V4 签名、避免敏感文件 `public-read`、可用 Referer 防盗链和 CDN 私有回源降低暴露。[来源: https://www.alibabacloud.com/help/en/oss/user-guide/how-to-obtain-the-url-of-a-single-object-or-the-urls-of-multiple-objects]

腾讯云 COS 支持上传和下载预签名 URL，可用签名有效期控制链接有效时间；控制台可快速生成 1 小时临时链接，COSBrowser 临时链接最长取决于账号类型。[来源: https://www.tencentcloud.com/document/product/436/45243] COS 访问日志可记录源 bucket 的访问细节，并可用于查询上传、下载、删除等操作；腾讯云还建议异常流量场景下启用访问日志、防盗链和告警策略。[来源: https://intl.cloud.tencent.com/document/product/436/46377]

### 3.3 DLP、DRM 与外发追踪

DLP 负责发现、阻断和审计敏感内容外流；Google DLP 可审计敏感内容使用、阻止外部分享、向用户警告、向管理员告警，并通过安全调查工具处理事件。[来源: https://support.google.com/a/answer/9646351] Microsoft Endpoint DLP 可配置云外发限制、受限应用、浏览器和域限制，并默认可审计 Office、PDF、CSV 文件活动。[来源: https://learn.microsoft.com/en-us/purview/dlp-configure-endpoint-settings]

DRM/IRM 适合“文件离开系统后仍受控”的场景。Microsoft 说明 SharePoint IRM 是较旧的文档库级下载保护技术，而敏感度标签同时提供视觉标记、加密和更完整的使用权限控制。[来源: https://learn.microsoft.com/en-us/purview/sensitivity-labels-sharepoint-onedrive-files] 但 DRM/IRM 会带来文件类型、客户端支持、协同编辑、检索和外部用户兼容性约束，不能作为一期唯一方案。[来源: https://learn.microsoft.com/en-us/purview/sensitivity-labels-sharepoint-onedrive-files]

## 6. 推荐落地方案

### 6.1 一期可做

1. 文件存储默认私有，业务系统只保存原始文件 URI，不向前端暴露原始对象地址；所有访问先过应用层权限校验，再生成带水印预览或短时下载链接。[来源: https://www.alibabacloud.com/help/en/oss/user-guide/how-to-obtain-the-url-of-a-single-object-or-the-urls-of-multiple-objects]
2. 服务端生成带水印 PDF/图片副本，水印内容至少包含用户姓名/账号、部门、访问时间、文件编号或资质编号；高敏文件可采用栅格化水印以提高移除成本。[来源: https://support.box.com/hc/en-us/articles/4404822772755-Enterprise-Settings-Content-Sharing-Tab]
3. 带水印预览与无水印下载分离：预览只给带水印副本；无水印必须提交用途、项目/客户、文件清单、期望有效期，经钉钉审批后生成短时链接。
4. 审批通过后生成无水印文件派生件和短时预签名 URL；URL 默认允许多次访问到期，应用侧记录首次访问和每次访问，是否限制次数由客户确认。[来源: https://docs.aws.amazon.com/AmazonS3/latest/userguide/using-presigned-url.html]
5. 下载审计字段最少包含：申请单 ID、审批单 ID、资质 ID、文件版本、申请人、审批人、下载人、下载时间、IP、User-Agent、水印策略、链接过期时间、对象存储 request id。
6. API 获取文件必须与页面下载同一套授权规则：投标助手等下游系统只能拿带水印文件或经审批的短时无水印链接。

### 6.2 二期增强

1. 一次性链接：引入下载 token 表，审批后发 token，token 兑换一次极短时预签名 URL 后立即失效；失败重试需要幂等窗口。[来源: https://aws.amazon.com/blogs/storage/implement-single-exchange-tokens-for-short-lived-amazon-s3-presigned-urls-with-terraform/]
2. DLP/IRM：接入终端 DLP、网关 DLP 或 Microsoft/Google 类似能力，覆盖“下载后复制、打印、上传外部网站、邮件外发”等行为。[来源: https://learn.microsoft.com/en-us/purview/dlp-configure-endpoint-settings]
3. 风险策略：按文件敏感级别、用户角色、部门、外部项目、下载频率触发二次审批或安全提醒。
4. 外发追踪：水印加入审批单号、用途、客户名和唯一追踪码；对象日志和业务审计进入统一检索。
5. 预览沙箱：高敏资质仅允许在线预览，禁止浏览器下载原始文件，必要时降低分辨率或分页渲染。

### 6.3 暂不建议

1. 不建议一期上完整企业 DRM/IRM，自研兼容 Office/PDF/图片、外部客户打开、撤销权限和离线阅读成本高。[来源: https://learn.microsoft.com/en-us/purview/sensitivity-labels-sharepoint-onedrive-files]
2. 不建议把敏感资质对象设为 `public-read` 或长期公开 URL；阿里云文档明确提示 public-read 会让任何拿到 URL 的人访问，且存在流量滥用风险。[来源: https://www.alibabacloud.com/help/en/oss/user-guide/how-to-obtain-the-url-of-a-single-object-or-the-urls-of-multiple-objects]
3. 不建议只做前端水印或只做静态水印；前端叠加容易绕过，静态水印难以追踪到具体外泄人。
4. 不建议一期强承诺“一次性下载链接”除非实现 token 状态机；对象存储预签名 URL 本身通常可重复访问至过期。[来源: https://docs.aws.amazon.com/AmazonS3/latest/userguide/using-presigned-url.html]

## 11. 对 PRD / 架构文档的输入

### PRD 输入

- 权限动作拆分为：查看列表、查看详情、在线预览带水印、导出带水印、申请无水印、审批无水印、下载无水印、重新申请、撤销链接。
- 无水印下载申请单字段：申请人、部门、资质文件、文件版本、用途、项目/客户、期望有效期、期望次数、审批链、审批意见、过期时间。
- 审计页面需求：按用户、文件、申请单、审批人、下载 IP、时间范围、是否过期、是否失败筛选。
- 水印策略配置：水印文本变量、透明度、位置、旋转角度、是否栅格化、适用文件类型、敏感级别。
- 预览与下载隔离：预览不暴露原始对象地址，预览资源默认带水印，下载必须走审批和签名链接。

### 架构输入

- 文件服务分层：原始文件、带水印派生文件、无水印临时派生文件、预览缓存、审计日志分表。
- 授权链路：Logto 身份 → 业务角色/部门权限 → 文件可见性 → 下载审批状态 → 对象存储短时 URL。
- 签名链接策略：对象私有、短有效期、HTTPS、最小权限、生成前业务校验、生成后访问审计。
- 一次性下载预留：审批 token 表、兑换状态、幂等窗口、失败重试、过期清理任务。
- 日志关联：应用审计日志必须记录对象 key、request id 或签名 token id，以便与对象存储访问日志对齐。
- 安全测试：越权查看、越权下载、链接过期、重复下载、审批绕过、前端绕过水印、跨部门搜索泄露、API 文件越权。
